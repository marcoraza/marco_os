Fundamentos

Visao Geral

O Marco OS e o teu sistema operacional pessoal. Um lugar unico onde todos os aspectos da tua vida se encontram.

psychology

O Que E o Marco OS

Sistema Operacional Pessoal

Um dashboard unificado que centraliza todas as areas da tua vida: financas, saude, aprendizado, projetos, network, notas e agentes de IA. Nao e so uma colecao de dados — e um sistema inteligente que te ajuda a tomar decisoes melhores atraves de contexto e automacao.

problem

Por Que Estamos Fazendo Isso

O Problema

Informacao espalhada: Notion, planilhas, apps diversos, notas soltas. Nada conversa entre si.

Falta de contexto: Ver um numero isolado nao te diz nada. "Gastei 800" sem saber quanto tens e inutil.

Decisoes fragmentadas: Sem uma visao unificada, cada decisao e tomada no vacuo.

Arquitetura em 3 Fases

Fase 1
Notion + JSON
Usar o Notion como backend, sincronizar via API, frontend renderiza JSONs.
Fase 2
Hybrid
Dados criticos em Supabase, resto no Notion. Onboarding conversacional ativo.
Fase 3
Full Supabase
Banco proprio, real-time, mobile app nativo, sincronizacao cloud completa.

Configuracao

CLAUDE.md Sugerido

Regras operacionais para o Claude Code baseadas nos padroes de fricção identificados. Copie e cole no seu CLAUDE.md.

Deployment Checklist

Antes de qualquer push
## Deployment Checklist - Verify all env vars are set in production - Never push without confirming with user first - After deploy, confirm live URL is responding - Run smoke test on critical paths before declaring done

Debugging Rules

Limites de tentativas
## Debugging Rules - After 2 failed attempts on the same bug, STOP and ask - Never blame cache without evidence (check logs first) - Diagnose root cause before proposing a fix - Summarize your diagnosis in 1 sentence before coding

VPS/Infrastructure

Ambiente de producao
## VPS/Infrastructure - Check for orphan processes FIRST before starting services - SSH commands timeout at 30s — use background jobs for long ops - Always run smoke test after any infra change - Confirm ports are open before declaring service is up

Payment Integration

Fluxos financeiros
## Payment Integration - Pix is not a subscription — they are different flows - Map all architectural differences BEFORE writing any code - Never assume payment provider behavior — read the docs first - Test with sandbox credentials before any production deploy

Testing

Qualidade do codigo
## Testing - Run full test suite after any refactor - grep for removed function names in test files before deleting - A passing build does not mean the feature works - Confirm test coverage on the changed path explicitly

UI/Frontend

Mudancas visuais
## UI/Frontend - Describe proposed visual changes and ask for confirmation before implementing - Never declare done based on build success alone — check in the browser - Screenshot or describe the result after UI changes - Ask if the interaction feels right, not just if it compiles

Autonomous Deploy-Test-Fix Loops

Ciclos automatizados
Deploy the feature, run the test suite, interpret any failures, fix them, and redeploy. Repeat until all tests are green. Maximum 5 cycles. If not green after 5 cycles, stop and report what's blocking.

Parallel Multi-Agent Codebase Ops

Workstreams paralelos
Split this batch of tasks into independent workstreams with no shared files. Assign each to a parallel agent. Once all are complete, coordinate the merge and resolve any conflicts before presenting the final result.

Pre-Flight Environment Validation

Checagem pre-deploy
Before any deploy, validate: open ports, git state (clean branch, no uncommitted changes), VPS connectivity, all required credentials present, deploy targets reachable, and database migrations are up to date. Report any blockers and stop if critical issues are found.
